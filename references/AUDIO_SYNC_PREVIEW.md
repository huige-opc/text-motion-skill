# 音画同步预览方案

当需要快速预览 SRT + 音频的同步效果，而不走完整渲染流程时，可以用这个轻量方案。

## 原理

1. 把 SRT 段按内容分组映射到场景
2. 播放音频时按 `currentTime` 实时计算当前应显示的场景
3. 浏览器渲染，OBS 录制

## 关键文件

### 内置脚本

| 文件 | 作用 |
|---|---|
| `scripts/build-timing-from-srt.mjs` | SRT 按内容/时长 → 场景时间轴 JSON（技能内建） |

### 项目内文件（参考 `projects/huige-ai-video/v2/`）

| 文件 | 作用 |
|---|---|
| `audio.mp3` | 录制的口播音频 |
| `audio.srt` | 修正后的字幕 + 时间戳 |
| `scene-timing.json` | 生成的场景时间表 |
| `index.html` | 音频驱动播放器 |
| `assets/main.js` | 核心同步逻辑 |

## 播放器模板

`references/audio-sync-player-template.js` 是一个即用型播放器模板，包含：

- 音频驱动的场景切换
- 键盘导航（空格暂停/播放、方向键切换场景、Home/R 重置）
- 自动播放（被浏览器拦截时点击激活）
- HTML 模板参考（注释中）

将模板复制到项目后，搭配 `scripts/build-timing-from-srt.mjs` 生成的 `scene-timing.json` 即可运行。

## 适用场景

- 录制前验证口播稿与画面匹配度
- 不需要精确到词的时间轴对齐
- 快速制作粗剪版本
