# 口播稿写作规则（text-motion-skill 内嵌完整版）

## 核心准则

口播稿不是文章。文章是给人看的，口播稿是让人说的。每句话写完之后，默读一遍——说出来顺不顺，不顺就改。

---

## 真人口述表达方式（强制）

写口播稿时，必须优先使用"真人对着镜头讲出来"的表达方式。不要把文章句子改短就当口播，要像一个人现场分享自己的真实感受。

### 口播表达的核心标准

| 要写成 | 不要写成 |
|-------|---------|
| 具体场景 + 真实反应 + 一句自我判断 | 抽象观点 + 总结判断 + 书面解释 |
| 晚上看书，看到某段话，很有感触；第二天一忙，啥都想不起来了 | 晚上看书觉得这句话说得太对了；第二天脑子里啥也没剩下 |
| 说白了，只是当时感动了一下自己 | 本质上只是产生了短暂情绪共鸣 |
| 把书里的东西，拉回到现实里 | 把书里的知识应用到自己的具体问题场景里 |
| 帮你把读过的书，真的用上 | 帮你完成从阅读到应用的转化 |
| 看到一个概念没看懂，别硬扛，直接问它 | 遇到不理解的概念，可以向AI提问 |
| 不知道怎么处理，就让它拿书里的方法陪你一起分析 | 在现实问题中调用书中方法论辅助决策 |
| 同一个问题，让不同作者各说各的，你一听就知道差别在哪 | 将不同书籍观点进行横向对比分析 |
| 心里烦，事情也乱，不知道该怎么处理 | 面对复杂问题和负面情绪时需要解决方案 |
| 它没上来劝我，也没列123，而是讲了个普通人的小故事 | 它用作者的思维和风格帮我解决问题、解决情绪、获得价值 |
| 不是给我标准答案，而是让我换了个角度看这件事 | 真正帮助用户获得深层价值 |

### 改写规则

- 先问：这句话像不像真人嘴里说出来的？如果不像，重写。
- 多用"我以前也这样""你有没有这种感觉""比如""说白了""后来我发现""我听完第一反应是"这类自然转场。
- 多写具体动作：晚上看书、第二天一忙、看到某段话、直接问它、别硬扛、打开就能用。
- 多写真实反应：很有感触、啥都想不起来、当时感动了一下自己、这味儿就对了。
- 少用抽象词：应用、转化、赋能、能力沉淀、知识体系、方法论、具体问题场景、核心价值、参与到问题里。
- 少用书面结构词：首先、其次、因此、由此可见、换句话说、从本质上讲。
- 功能点也要口语化。比如不说"选书/用书/答疑"，优先说"帮你挑书 / 陪你读 / 帮你想怎么用 / 读到哪儿卡住了直接问"。
- 可以保留结构，但结构不能露出来。正文要像聊天，不像提纲。

### 口播句子自检

写完每一段，都按下面三问检查：
1. 我自己念出来顺不顺？
2. 这句话是不是普通人平时会这么说？
3. 有没有更短、更生活、更像嘴里冒出来的说法？

如果答案是否定的，优先按真人口述改写，而不是按文章逻辑补充解释。

## 通用价值框架（四层，缺一不可）

不管走类型A还是类型B，一条口播稿都要覆盖下面四层。**缺任何一层，观众都可能中途划走。** 写完后逐层自检：

| 层次 | 核心问题 | 对应位置 |
|------|---------|---------|
| 为什么需要知道 | 观众看完开头3秒，觉得"这跟我有关"吗？ | 开头 |
| 为什么听你讲 | 稿子里有你的真实经历/踩坑/实测吗？还是干巴巴搬运？ | 主体 |
| 为什么这个知识是对的 | 例子贴切、逻辑自洽、有真实效果或案例佐证吗？ | 解释/演示部分 |
| 为什么此刻行动 | 结尾有没有具体动作让观众现在就能做（收藏/关注/评论）？ | 结尾 |

**自检要点：**
- 第②层最容易被砍——一压字数就把"我自己怎么用、踩过什么坑、实测什么结果"删掉，只剩"我做了个东西"。宁可砍别的，也要留住这一层，它是你和搬运号的区别。
- 第③层不是讲道理，是给画面：选大众熟悉的场景、给真实效果、举贴切的小例子（详见各类型的例子规范）。
- 四层不是四个段落，可以交叉。比如一句"我试过让它演刘震云聊中年失业，当场就愣住了"同时补齐第②层（真实经历）和第③层（贴切例子）。

> 这套四层和类型B的「销售四问」（为什么需要买→为什么跟你买→为什么买这个→为什么此刻买）同源：销售四问偏成交转化，四层价值框架偏内容价值，任何一条口播稿都应先过这四层，再按类型套具体结构。

## 第一部分：辉哥写作风格

### 1.1 风格本质

**核心就一句话：像在跟朋友分享，不是在讲课。**

#### 核心思维方式（最重要）

写每一句话之前，先站在观众角度问自己：**"我要是观众，看到这句话会怎么想？"**

- 他会觉得"这跟我有关" → 留着
- 他会觉得"这有意思" → 留着
- 他会觉得"这有用" → 留着
- 他会觉得"你是谁，凭什么" → 这句有问题，改
- 他会觉得"所以呢？然后呢？" → 上下衔接断了，补逻辑

**你之所以觉得某句话"突兀"，就是因为它没有回答观众心里那个"然后呢"或者"凭什么"。**

#### 自然逻辑链（工具分享类）

你对工具分享类内容有一个自然的逻辑顺序，不能乱：

```
痛点 → 反常识/原理 → 工具是什么 → 觉得有用 → 收藏 → 上实操
```

每一步都必须回答了观众心里的一个问题：
1. **痛点** → "这跟我有关"
2. **反常识/原理** → "原来是这样"
3. **工具是什么** → "知道是什么东西了"
4. **觉得有用** → "这个对我有用"
5. **收藏** → 这时候喊收藏，观众才愿意行动
6. **上实操** → "看看怎么用"

这6步中间不能跳步。比如痛点还没说完就直接喊收藏，观众反应是"你是谁，凭什么"。

#### 选词逻辑

你不是固定用某些词，而是倾向于选**更口语、更有画面感**的那个选项：

| 你倾向 | 而不是 |
|-------|-------|
| 有画面感的动词：糊、摘、搓、卡、加上去 | 书面动词：提取、获取、覆盖、添加 |
| 短词：搞定、完事 | 长词：解决、完成 |
| 有情绪的词：烦、省得 | 中性词：困扰、避免 |
| 直接说结果：不用会员、不用PS | 绕弯子：无需额外付费 |
| 简单收尾：完事 | 啰嗦收尾：这样就完成了 |

**所以不是背词表，是想表达一个意思时，选更短、更直接、更有画面感的那个。**

#### 语言细节（从实际修改中提炼）

这些是你写稿时下意识会调整的地方，也是判断"顺不顺"的标准：

| 场景 | 你倾向 | 不倾向 |
|------|-------|-------|
| 动作主语 | 「自动给摘出来」去掉"你"更干脆 | 「自动给你摘出来」 |
| 节奏助词 | 「不用会员，也不用PS」加"也"有停顿感 | 「不用会员，不用PS」有点赶 |
| 口语衔接 | 「我就直接手搓了一个」加"就"更自然 | 「我直接手搓了一个」少了点过渡 |
| 收尾补充 | 「后面我会分享更多的干货」加"我会"更完整 | 「后面分享更多干货」有点秃 |
| 收藏指令 | 「先收藏，省得回头找不到」直接、不用"建议" | 「建议先收藏起来」太软 |
| 形容词具体化 | 「大水印」一个字画面感更强 | 「水印」太普通 |

#### 句子节奏

- **短句为主**，一句说一个事。长句拆碎。
- 关键信息单独成句，不藏在从句里。收藏指令尤其要独立成行。
- 用问句拉近距离，不是真让观众回答，是让他心里默认。
- 重复重要信息，不怕啰嗦，怕他没听到。

#### 开头逻辑

你喜欢用开头直接给观众一个**理由留下来**，看你选哪类内容：

**类型A（工具分享类）：**
- **收藏理由**：「先收藏，省得回头找不到」——这个有用
- **身份反问**：「你是不是也……」——这跟我有关

**类型B（观点输出类）：**
- **反常识型**：「[常见认知]？其实是[反常识结论]」——这个跟我想的不一样
- **扎心型**：「你是不是也以为[旧认知]是对的？」——说的就是我

#### 连接逻辑

- 「其实」——后面接反常识/真相
- 「搞清楚这个」——后面接解决方案
- 「来，我们直接上实操」——从理论切换到操作
- 「好了」「搞定」——一段结束，干净收尾

#### 辉哥语气词库（自然融入）

这些语气词要自然，不要为了加而加：

| 语气词 | 适用场景 | 示例 |
|-------|---------|------|
| **呢** | 缓和语气、引出话题 | 「大部分同学呢」、「我就干脆仔细研究了一下呢」 |
| **啊** | 提醒注意、强调 | 「面板显示位置啊」、「要不要弹登录窗啊」 |
| **吧** | 商量、确认 | 「能用就行吧」、「分清楚了就简单了吧」 |
| **啦** | 惊喜、轻松 | 「改一处全生效，是不是非常NICE？」 |
| **了** | 完成、结束 | 「改完了」、「搞定了」 |

#### 辉哥口语化表达（优先选用）

有画面感、有情绪的词，比书面词效果好：

- 说实话、我估计、说白了、干脆、心累、费劲巴拉、NICE、清清爽爽、搞定
- 有画面感的动词：糊、摘、搓、卡、加上去
- 有情绪的词：烦、省得
- 直接说结果：不用会员、不用PS

#### 自称与称呼

- **自称**：辉哥
- **受众称呼**：大家、朋友们、同学


### 1.2 风格清单

| 要做的 | 不要做的 |
|--------|---------|
| 短句，一句话一个意思 | 长句、多层嵌套 |
| 口语化，默读顺口 | 书面语、论文腔 |
| 有情绪的干货 | 干巴巴的知识点 |
| 「我」的视角分享 | 无来源的客观陈述 |
| 直接给结论 | 铺垫太长、讲半天没重点 |
| 实操导向，「来，看操作」 | 空谈理论不落地 |

### 1.3 结构节奏

**先判断类型，再选结构。** 口播稿按内容性质分两类，结构不同，不能混用。

| 特征 | 类型A：工具分享/教程类 | 类型B：观点输出/认知颠覆类 |
|------|----------------------|--------------------------|
| 目的 | 教会一个操作/工具 | 改变一个认知/观念 |
| 核心 | 怎么做 | 为什么你以为的是错的 |
| 观众获得感 | 「学会了」 | 「原来是这样」 |
| 钩子逻辑 | 这个有用 | 这个跟你想的不一样 |
| 情绪主线 | 实用感 → 成就感 | 认知冲突 → 恍然大悟 |
| 观众看完 | 收藏了，下次照着做 | 转发了，跟朋友讨论 |
| 适合场景 | 软件教程、工具测评、操作演示、技巧合集 | 观点输出、行业吐槽、认知类、经验感悟、避坑指南 |
| 开口清单 | 想教别人怎么干的时候 | 想纠正别人认知的时候 |

**两句话帮你快速判断：**
- **你要教别人怎么用一个东西？** → 走类型A
- **你要告诉别人"你的想法是错的"？** → 走类型B
- 如果拿不准，看观众看完最可能干什么：**收藏**走A，**转发/评论**走B

**写法差异速览：**

| 对比项 | 类型A | 类型B |
|--------|-------|-------|
| 正文节奏 | 一气呵成，操作步骤连贯不打断 | 一拳一脚，每段打一个点，中间有递进 |
| 语言风格 | 短促有力，指令明确 | 语气词多，像在聊天，用反问拉近距离 |
| 过渡信号 | 「来，直接上实操」「第一步」 | 「好，第二个」「第三个更常见了」 |
| 案例要求 | 展示操作过程和真实效果 | 选大众熟悉的场景让观众代入 |
| 金句位置 | 结尾一句总结升华 | 每个误区一句，结尾排比收束 |
| 情绪驱动 | 奖励期待 > 打断预测 | 打断预测 > 精准命名 > 损失厌恶 |
| 结尾转化 | 收藏引导优先 | 关注/讨论引导优先 |

#### 类型A：工具分享/教程类

常见节奏形态（可组合）：
- **快节奏**：钩子 → 直接演示 → 收尾
- **讲解型**：痛点 → 原理一句话 → 方案 → 演示 → 收尾
- **故事型**：我的经历/发现 → 怎么做 → 效果如何
- **短平快**：直接说事 → 完事
- **问题驱动**：提问 → 解答 → 总结

#### 辉哥标准叙事流程

如果是教程/经验分享类，优先用这个结构。每步要有画面，要有一句能记住的话。

**1. 痛点引入 — 「你是不是也……」**
上来就戳一下，不铺垫。第一句话让观众觉得"说的就是我"。
> 金句收尾：「你不是手残，是没人给你讲清楚。」

**2. 个人经历 — 「我以前也……」**
用自己的故事建立信任，别讲大道理。讲当时多烦、多折腾、多蠢。
> 金句收尾：「踩过的坑，都是学费。」

**3. 解决方案 — 「后来我发现，其实……」**
一句话说透核心方案，不要绕。让观众觉得"原来这么简单"。
> 金句收尾：「搞懂这个，后面就全是重复劳动。」

**4. 分点说明 — 「来，就三步。」**
拆成123，每一步都用口语信号引导（「第一步」「然后」「最后」）。每步之间给观众喘口气，别一口气倒完。
> 每步做完加一句效果：「看到了吗？」「这就搞定了」「是不是很简单？」

**5. 总结升华 — 「所以你看……」**
从工具/方法拉升到通用思路，让观众觉得"我不光学了这个工具，我还懂了一个道理"。
> 金句模板：「[工具]是表象，[思路]才是本质。XX，日积月累，也挺可观的。」

#### 类型B：观点输出/认知颠覆类

**核心逻辑**：先打破旧认知，再填入新框架。不是教人怎么做，而是让人"意识到自己错了"。

**完整结构（8步循环）：**

| 步骤 | 功能 | 时长占比 | 示例 |
|------|------|---------|------|
| 1. **反常识Hook** | 制造认知冲突，让人停下来 | 前3-5秒 | 「90%直播间不变现，就是因为干货讲太多」 |
| 2. **立权威** | 快速说明凭什么你说这话 | 10-15秒 | 「做了4年IP，200万粉丝，培训上千老板」 |
| 3. **打认知误区** | 指出旧认知错在哪，制造认知空缺 | 30秒 | 讲干货只有两种结局，都是死路 |
| 4. **重新定义目标** | 以终为始，倒推起点 | 15秒 | 「干货不是目的，变现才是」 |
| 5. **递进给框架** | 抛出结构化方法论，填认知空缺 | 40秒 | 销售四问→需求三要素→131模型 |
| 6. **案例佐证** | 每个框架配具体案例，让观众代入 | 30秒 | 电钻与孔、美容院老板娘 |
| 7. **升维收尾** | 从技巧拔高到底层逻辑 | 15秒 | 「兴趣电商的本质是制造需求」 |
| 8. **引流/转化** | 软性引导下一步 | 5秒 | 「预约我的直播来连麦」 |

**3个核心框架（可直接套用）：**

**框架1：销售四问**
```
为什么需要买 → 为什么跟你买 → 为什么买这个 → 为什么此刻买
```
- 成交的底层逻辑，任何产品适用
- 一篇口播稿至少覆盖前三个，逼单环节覆盖第四个

**框架2：需求三要素**
```
利益（痛点唤醒）× 认知（重要性排序）× 价值观（态度吸引） = 需求
```
- **利益**：不是卖产品，是卖解决方案。用户要的是孔，不是电钻
- **认知**：改变认知→改变行为→带来结果。如美容院先做复购还是先学小红书
- **价值观**：活出态度，吸引同频的人。无关对错，是喜好

**框架3：131循环模型**
```
1个自我介绍（为什么跟我买）
+ 3个干货结构（为什么需要买）
+ 1个逼单话术（为什么买这个 + 为什么此刻买）
= 20分钟一个完整销售循环（直播）/ 2-3分钟一条口播（短视频）
```
- 直播间每20分钟一个循环，每个循环内必须完成4个"为什么"
- **短视频版**：压缩为2-3分钟，结构一样，但每个干货点精简为一拳一脚，直接亮痛点+反认知

**写法要点：**
- 每句话都在打认知差，不要在已知信息上停留
- 框架要有名字（销售四问、131模型），方便记忆和传播
- 打误区时态度要笃定，不能"可能不对"，要"你就是错的"
- 案例选大众熟悉的场景（电钻、美容院），不选冷门领域
- 升维收尾让观众觉得"这人确实有高度"，不是单纯的销售

**实战经验（从实际输出中提炼）：**

*关于例子*
- 例子要选**当下热门**的、普通人日常遇到的场景，别用五笔字型这类太老的
- 比如：写周报、改文案、转录音、做小红书封面、写朋友圈 — 这些都是普通人真会干的事
- **避免生僻工具名**（Sora、Grok这类），直接说场景就行

*关于口语*
- 避免书面词：不说"心智模式""消费行为""术与道"，说"找按钮vs提需求""跟刷短视频没区别""皮毛vs根本"
- 衔接过渡要变化，**不能重复用同一句式**（比如"但你有没有想过"用了5次就很模板化）
- 语气词（呢、啊、吧、嘛）要自然地加进去，一段话里分布均匀

*关于结构*
- 每个误区/干货点**直接亮痛点**，不铺垫。第一句就让观众觉得"说的就是我"
- **每个误区末尾加一句金句收住**，不只结尾才有。金句格式参考：「xx是最容易xx的，也是最容易xx的」「你追的不是xx，是xx」「xx救不了xx的人」
- 用**反问句式**拉近距离：每个误区结尾加一句「是不是你？」让观众自我代入
- 用**场景代入句式**引导回想：「你回想一下，是不是有过这种经历」「你是不是也这样」
- 多个误区之间**建立递进关系**，不要平行排列。从表到里层层深入，比如：方向错了→方式错了→心态错了→用法错了→根上错了。每段开头加递进提示：「第一个，表面上的」「第二个，再深一层」「第三个，再往深了挖」
- **每个误区配一个画面感场景**，让观众能代入。比如「花99买了课第二天全忘了」「收藏了再也没打开过」「群里求提示词复制过来不是一回事」。不要只讲道理，要给画面
- 结尾要有**金句总结**，最好排比句式（"工具会过时，框架不会/提示词会过时，审美不会/操作会过时，判断力不会"），让观众能记住
- **升维拔高收尾**：最后一段要从具体话题拉升到底层逻辑或时代趋势，让观众觉得"这人确实有高度"。比如从AI学习拉升到"AI时代最残酷的不是工具变得快，是你的思维方式跟不上"
- **逼单要有紧迫感**：行动号召加时间压力，如「不是明天啊，是现在」「现在就去做」
- 轻引流根据场景调整：短视频→"关注我，后面继续拆更多干货"；直播→"预约我的直播来连麦"

#### 辉哥过渡短语库

**类型A（工具分享类）：**
- 「那到底怎么弄呢？」
- 「说起来也不复杂」
- 「我们来看一下」
- 「理清了这个思路」
- 「如果你也卡在这」
- 「最后说一句」

**类型B（观点输出类）：**
- 递进切换：「好，第二个」「第三个更常见了」「最后一个」
- 表层→深层：「好，这三个还算浅的。还有两个更深的」
- 误区→方案：「好，五个坑都说完了。那正确做法到底是啥呢？」
- 引出案例：「给你说个真事啊」
- 引出总结：「所以你看啊——」
- 人设收尾：「好了，我是辉哥」


#### 中间结构的起伏

**口播稿最怕中间平。** 开头钩子做好了，但中间全是操作步骤，观众看着看着就刷走了。

实操部分要有节奏变化，不能一路平铺：

| 段落 | 节奏 | 情绪 | 口语信号 |
|------|------|------|---------|
| 安装步骤 | 快，一气呵成 | 紧凑 | 收尾「搞定」「够简单吧？」 |
| 效果展示 | 停一下，让观众看画面 | 惊喜 | 「点开——看到了吗？」 |
| 功能展开 | 平稳，说清楚就行 | 实用 | 正常语速介绍 |
| 额外价值 | 略提一口气 | 意外收获 | 「还有一个很好用的」 |

关键原则：
- 每个段落之间有情绪变化，不能一个调到底
- 用口语信号做段落切换（「来，我们直接上实操」「还有一个很好用的」），不用分隔线
- 口播稿是念给人听的，任何视觉符号（---、**、01标记）在正文中都没意义。正文=只放念出来的话

#### 过渡信号

过渡不是靠视觉分隔，是靠口语信号自然切换：

| 场景 | 信号 | 作用 |
|------|------|------|
| 理论→实操 | 「来，我们直接上实操」 | 提醒观众注意，接下来要看操作 |
| 操作→效果 | 「看到了吗？」 | 把观众注意力拉回画面 |
| 功能→新功能 | 「还有一个很好用的」 | 暗示还有价值，别划走 |
| 一段结束 | 「搞定」「好了」 | 干净收束，不拖泥带水 |

## 第二部分：短视频爆款方法论

### 2.1 树形结构

内容像一棵大树，必须层次分明：

| 层级 | 内容 | 作用 | 时长占比 |
|------|------|------|---------|
| 树根 | 问题/痛点 | 开头抛出，让用户觉得"这跟我有关" | 前3-5秒 |
| 树干 | 核心观点 | 一句话说清楚要讲什么 | 5-10秒 |
| 树枝 | 理由/分论点 | 用实操步骤做理由，层层展开 | 20-40秒 |
| 树叶 | 证据/效果 | 展示真实效果，让用户信服 | 10-15秒 |

### 2.2 四维情绪驱动（每3句必自检）

| 情绪 | 说明 | 示例 |
|------|------|------|
| 打断预测 | 反常识/反直觉，制造认知冲突 | 「豆包那个水印，不是原图带的」 |
| 损失厌恶 | 制造恐惧，让人害怕错过/吃亏 | 「开会员没必要，自己P不会搞」 |
| 精准命名 | 替用户说出心里话，建立同盟 | 「那个水印真的很烦」 |
| 奖励期待 | 描绘爽点，让人想看下去 | 「一个小工具就解决了」 |

### 2.3 开头3秒（完播率核心）

前3秒划走率超过70%。开头必须做一件事：让观众停下来。

**不是所有开头都要走同一个路子。根据内容性质选型：**

| 开头类型 | 适合场景 | 示例 |
|---------|---------|------|
| **收藏型** | 工具分享、教程类 | 收藏放在方案之后、实操之前，不放在开头第一句。详见下方收藏指令位置说明 |
| **反常识型** | 有反认知亮点的内容 | 「豆包那个水印，不是原图带的」 |
| **扎心型** | 痛点强烈的选题 | 「你是不是也遇到过这种事……」 |
| **问题型** | 问答、观点类 | 「豆包生图有水印怎么办？」 |
| **结果型** | 效果展示类 |先展示结果，再说怎么做 |
| **故事型** | 个人经历类 | 「我最近发现一个事……」 |

核心要点：
- 第一句给价值——要么说中痛处，要么抛出反常识。让观众觉得"这跟我有关"或"这个有意思"
- **收藏指令的位置**：不要放在开头第一句，也不要贴在方案后面当尾巴。正确位置是：观众知道是什么工具、觉得有用之后，再单独成句。顺序是 痛点→原理→工具→有用→**收藏**→实操
  - 观众心理：先得到价值，才愿意行动
  - 如果第一句就喊「先收藏」，反应是"你是谁，凭什么"
  - 如果贴在方案后面当尾巴，反应是"怎么突然跳到收藏了"
  - 收藏单独成行，前后留气口，不跟其他信息挤在一起
- 不要铺垫，不要绕，第一句给信息量或者给情绪

### 2.4 爆款标题

标题负责让人点进来。写标题不是写概括，是写"不点开你就亏了"。

**直接套用下面的模板，填你的内容就行。**

#### 8个可套用的标题模板

| # | 公式 | 套用举例 |
|---|------|---------|
| 1 | **[痛点]？一个[工具/方法]搞定** | 豆包生图有水印？一个插件搞定 |
| 2 | **[反常识信息]，不是[你以为的那样]** | 豆包那个水印，不是原图自带的 |
| 3 | **别再[错误做法]了，[正确方法更简单]** | 别傻傻开会员了，豆包去水印有更简单的方法 |
| 4 | **[数字]步/分钟/个[动作]，[结果]** | 两步搞定豆包去水印 |
| 5 | **[目标]不用[麻烦方式]，[简单替代]** | 豆包去水印，不需要会PS |
| 6 | **先收藏再说，[价值理由]** | 豆包去水印，先收藏省得回头找不到 |
| 7 | **[问题]到底能不能[结果]？** | 豆包的水印到底能不能去掉？ |
| 8 | **所有[人群]注意了！[关键信息]** | 所有豆包生图的朋友注意了！ |
| 9 | **[结果]！别[错误方式]，[行动指令]（建议收藏）** | 豆包生图去水印！别开会员，插件放评论区了（建议收藏） |

#### 写完标题自己审

- [ ] 别人看到标题，1秒内知不知道跟自己有关？
- [ ] 标题有没有一个"非得现在看"的理由？
- [ ] 如果我是观众，我会不会想点？
- [ ] 标题和封面图说的是同一件事吗？
- [ ] 没有极限词（最、第一、国家级）？
- [ ] 承诺的事内容能对得上吗？

#### 平台差异

- **抖音**：前3-5字就要击中需求，短句强情绪
- **视频号**：价值感强一点，方便别人转发收藏
- **小红书**：带上关键词，像「教程」「干货」「亲测」

#### 技巧

- 标题10-25字，太长看不全
- 一条标题能用2个以上的模板组合更好
- **标题也要过极限词检测**——「终极」「最强」「全网第一」等词不仅在正文里禁用，标题里同样违规
- 写完默念一遍，念着顺不顺，不顺就是有问题

### 2.5 结尾

结尾是转化动作，至少设计以下两项：

**合规轻提示（重要）**
平台对站外引流和强转化管得严，口播里不要硬喊"私信我、加我、链接、我发给你、我发给你资料"。资料领取类内容优先用一句轻提示，点到为止：
- 「资料我已经整理好了。」

写法要求：
- 不解释"怎么拿"，懂得都懂。
- 不说"后面就不硬喊了"，这是多余解释。
- 不反复提醒，不在结尾再喊一遍。
- 不把领取动作说得太直白，避免平台判定强引流。
- 如果需要动作，只用平台内弱动作，比如"先收藏一下""需要的朋友留意一下"。

位置不要只放最后。更好的位置是：观众已经知道这个东西是什么、觉得有用之后，在中前段轻轻带一句"资料我已经整理好了。"；结尾只保留关注或继续分享，不再反复引流。这样既合规，也避免观众听到最后才觉得"原来你只是来要互动的"。

**收藏引导（最优先）**
收藏理由要合规，优先用"怕回头找不到"的理由：
1. 「先收藏，省得回头找不到」
2. 「这个后面你肯定用得上，先收藏一下」

收藏指令可以和插件位置合并在一起说，节奏更顺。

**关注引导**
「觉得有用的话，关注点赞，后面我会分享更多干货」

**评论引导**
开放问题效果不好，用户不知道回什么。**给具体选项：**
- ✅「是写文案还是做PPT？评论区告诉我，我挨个去研究」
- ❌「还有什么AI方面的工具，评论区聊聊」

## 第三部分：平台审核规则

### 3.1 绝对禁止内容

#### 极限词（禁词）

| 类别 | 禁用 | 替换 |
|------|------|------|
| "最"字类 | 最、最佳、最好、最大、最低、最新、最有效、最受欢迎 | 很、真的、不错、实用 |
| "一"字类 | 第一、全国第一、全网第一、唯一、独一无二 | 优选、好用 |
| "级/极"类 | 国家级、顶级、极品、极致、顶尖 | 专业、高质量 |
| 绝对表述 | 100%、绝对、永久、万能、无敌 | — |
| 虚假承诺 | 保证、包会、保过、无效退款 | 分享、推荐、仅供参考 |

#### 导流违禁

| 禁止 | 合规替代 |
|------|---------|
| 加微信、私信我、加我 | 关注公众号、看主页置顶 |
| 链接在评论区（直接说"链接"） | 插件我放评论区了（用"插件"替代"链接"） |
| 引导站外交易 | 平台内完成交互 |

### 3.2 AI生成内容标注（强制）

2025年9月起，所有平台强制执行《人工智能生成合成内容标识办法》：
- 所有AI生成内容必须显著标注
- 短视频发布前必须选择6类必选标签之一，否则无法发布
- 如视频中使用AI生成素材，必须在画面标注「内容由AI生成」
- 演绎类内容必须标注「虚构演绎，仅供娱乐」
- **后果**：未标注可导致限流、下架、封号

### 3.3 抖音平台红线

- 虚假夸张：功能描述必须有真实依据，不能夸大效果
- 博眼球/卖惨：禁止虚假演绎、自导自演
- 版权侵权：音乐、图片、视频必须有授权
- 刷量刷赞：禁止虚假互动

### 3.4 微信视频号红线

- 虚假摆拍：剧情演绎必须标注，否则认定为虚假博流量
- 2026年新规：已累计处置数千条违规内容
- 各类行业公约（金融、法律、房地产）持续收紧

### 3.5 2026年监管升级

- 方言谐音识别已上线，变声/谐音规避无效
- 画面文字OCR识别准确率99.2%（背景板文字也识别）

- 中央网信办13条负面清单

```
| 奖励期待 | 高/中/低 | — |
```

## 第七部分：text-motion-skill 专用两遍法

**text-motion-skill 场景下，口播稿必须走两遍。** 上面第六部分的通用格式属于「单遍稿」，用于直接录制。text-motion-skill 需要在录制之前先做视频画面规划，因此拆成两遍。

### 第一遍：详细口播稿（≈ 250-380 字/分钟）

**给 AI 做视频画面规划用。** 内容要饱满、有画面感、带场景划分。

每个场景必须标注：
- `场景NN [视觉类型] [锚点词]` — 锚点词 4-12 汉字，该场景独有，用于 SRT 自动匹配
- 视觉类型从预设清单选：钩子 / 痛点 / 数据 / 方案引出 / 演示 / 金句 / 对比 / 分类列举 / 场景切换 / 升华收尾

**关键原则**
- 每个场景对应一个视频画面切换（硬切）
- 场景内的文案保持连贯，可以是一句或多句
- 锚点词必须提炼自该场景的原文，不能凭空造
- 视觉类型从预设清单选，不要自己造

**预设视觉类型表：**

| 视觉类型 | 适用场景 | 示例 |
|---------|---------|------|
| 钩子 | 开头抓注意力 | "做视频的你，是不是也这样" |
| 痛点 | 描述用户困境 | "弄了一整天，效果还不如别人" |
| 数据 | 数字/统计信息 | "内置了 9 套视觉风格" |
| 方案引出 | 引出解决方案 | "我换了一套新方法" |
| 演示 | 操作/步骤说明 | "第一步读字幕，第二步分析" |
| 金句 | 一句话总结收束 | "AI 越用越懂你" |
| 对比 | 前后对比/对仗 | "不用打开任何软件" vs "一条命令搞定" |
| 分类列举 | 列举多个类别 | "第一类、第二类、第三类" |
| 场景切换 | 段落过渡 | "那这套 skill 到底适合什么人用？" |
| 升华收尾 | 结尾拉升高度 | "你越用越顺手" |

**输出格式：**

```
### 【详细口播稿（视频版）】

**类型：** A（工具分享/教程类） / B（观点输出/认知颠覆类）
**总字数：** ~2500 字
**视频时长参考：** ~10 分钟（按 257 字/分钟估算）

---

## 场景01 [钩子] [锚点: 口播录好了]

做视频的你，是不是也这样。口播录好了，配画面怎么办？

## 场景02 [痛点] [锚点: 弄了一整天]

打开剪辑软件，一堆轨道从哪开始？弄了一整天，效果还不如别人，干脆不想发了。

## 场景03 [方案引出] [锚点: 让 AI 直接帮我把字幕变成画面]

我最近换了一套新方法，专门解决这件事。让 AI 直接帮我把字幕变成画面，全程不用打开任何剪辑软件，就能生成一条带动画的视频。

（更多场景...）

---

### 【精简版参考数据】

| 场景 | 视觉类型 | 字数 | 预计时长 |
|------|---------|------|---------|
| 01 | 钩子 | 18 | ~4s |
| 02 | 痛点 | 28 | ~7s |
| 03 | 方案引出 | 42 | ~10s |
| ... | ... | ... | ... |
```

**锚点词选择标准：**
- 长度 4-12 个汉字
- 该场景独有的关键词（不会在其他场景重复出现）
- 优先选实词（名词/动词），避开语气词和过渡词
- `[锚点: ...]` 必须是该场景原文中出现的短语，用于后续 SRT 自动化匹配

### 第二遍：精简口播稿（≈ 200-260 字/分钟）

**给人录音 / TTS 配音用。** 砍掉所有视觉描述、场景标注、锚点词，只留念出来的正文。

用 `scripts/compress-script.mjs` 自动压缩，控制录音时长。

### 两遍稿关联

详细稿的每个场景锚点词 → 对应精简稿中的口播句子 → 对应 scene-timing.json 的 SRT 时间 → 对应场景 HTML 的 beats 数组

**详细稿决定画面，精简稿决定时间轴，二者通过锚点词一一对应。**

## 附：类型判断快速通道

拿到用户主题后，先做类型判断：

1. **用户要教一个操作/工具？** → 类型A，走「痛点→原理→工具→有用→收藏→实操」六步链
2. **用户要打破一个认知？** → 类型B，走 8 步循环（反常识Hook → 立权威 → 打误区 → 重新定义 → 给框架 → 案例 → 升维 → 引流）
3. **拿不准？** 看用户想让观众看完做什么——**收藏**走A，**转发/评论**走B

**text-motion-skill 项目普遍是「工具/技能介绍」，默认走类型A，但开头钩子可以借用类型B的反常识 Hook 增加打开率。**

## 第八部分：v2 血训沉淀（2026-07-21 迭代）

以下规则来自 text-motion-skill 介绍视频稿件的实战调试。每条都是踩过坑之后总结的**强制规则**，AI 必须严格执行。

### 8.1 开头 3 秒爆款钩子公式（AC 结合三合一）

**痛点铺垫式开头一律砍掉。** 3 秒内没数字、没反常识、没结果，观众直接划走。text-motion-skill 稿件的开头必须命中以下三合一公式：

```
[流程罗列制造对比感] + [反差数字 + 疑问] + [反常识补刀] + [产品动作] + [软性收藏]
```

**参考模板：**
> 一条 5 分钟的口播视频，从准备素材、录制、修改润色到剪辑，你猜现在需要多久？十分钟。而且我根本没打开剪辑软件。一段音频、一份字幕，直接出一条带动画的视频。做视频的朋友，这个技能可以先收藏。

**拆解：**
1. **流程罗列**「从素材、录制、润色到剪辑」— 让观众自动脑补工作量
2. **反差数字 + 疑问**「你猜现在需要多久？十分钟」— 数字冲突 + 悬念
3. **反常识补刀**「而且我根本没打开剪辑软件」— 打断预测
4. **产品动作**「一段音频、一份字幕，直接出一条」— 结果先亮
5. **软性收藏**「这个技能可以先收藏」— 平台内合规动作，不说"救你的命"

**禁用的开头模式：**
- ❌ 「做视频的朋友，你是不是也这样」（铺垫太软，3 秒抓不住）
- ❌ 「打开剪辑软件，一堆轨道从哪下手」（第二句才给信息，太晚）
- ❌ 「最烦的是时间轴地狱，300 多个关键帧」（用了行话，普通读者听不懂）

### 8.2 观众基础差 · 术语零容忍

**text-motion-skill 目标受众是「做视频的普通人」，不是技术圈。** 一切行业行话必须换成场景描述。

| 禁用（术语） | 必须替换成（场景动作） |
|-------------|---------------------|
| 关键帧 | 对着字幕一句一句往上卡文字 |
| 时间轴 | 盯着轨道 |
| 轨道 | 一条一条的字幕条 |
| 渲染 | 视频跑出来 |
| 分镜 | 画面切换 |
| 后期 | 剪视频的那一步 |
| SRT / 字幕文件 | 字幕（在读稿场景下不加后缀） |
| 特效 / VFX | 动画 / 画面变化 |

**判断标准：**「我妈能不能听懂？」听不懂就换。

### 8.3 平台合规 · 二级极限词库（扩充）

除官方极限词外，text-motion-skill 稿件还需清洗以下**隐形极限词**（一次审核踩过 3 次）：

| 隐形极限词 | 替换方案 |
|-----------|---------|
| 最爽的一点 | 特别爽的一点 / 用下来我觉得这点特别爽 |
| 最怕的就是… | 特别怕这种… |
| 最崩溃 / 最烦 | 特别崩溃 / 最烦这一步 → 卡在这一步反复内耗 |
| 真的能救你的命 | 这个技能可以先收藏 / 能帮你省下大把时间 |
| 秒杀所有… | 比…踏实多了 |
| 完虐 / 吊打 | 不用碰 / 不用打开 |
| 神器 | 技能 / 工具 |
| 逆天 / 无敌 / 王炸 | — |
| 保姆级 | 一步一步 |

**审核流程：** 稿件生成后必过一遍「最/第一/极/绝对/百分百/永久/包/保证/救命/神」扫描。

### 8.4 画面感改写清单（形容词 → 场景动作）

**规则：** 形容词是抽象的，场景动作是画面。每个形容词必须换成一个可以拍出来的动作或场景。

| 抽象形容 | 场景动作改写 |
|---------|------------|
| 简单 | 三样东西丢进去 / 你去泡杯咖啡回来就好 |
| 快 | 一杯茶还没凉透，初稿就出来了 |
| 慢 / 累 | 天没亮开机，天黑了还坐在那儿盯着轨道，脖子僵得转不动 |
| 智能 | 它抱着字幕从头听到尾 / 像有个隐形剪辑师连夜替你干完了活 |
| 温暖 | 米色文档、暖光、纸的味道 |
| 冷酷 | 冷冷的、几何感的、像杂志封面一样 |
| 科技感 | 屏幕黑得发亮、字带荧光 |
| 灵活 | 你就说一句话，整条视频的味儿就变了 |
| 一个人做很多事 | 选题、写稿、录音、剪辑一个人扛，凌晨两点还在电脑前改视频 |
| 追热点 | 热点今天中午冒出来，晚上就得发出去 |

**触发时机：** 每段文案写完自问「有没有形容词是可以拍出画面的？」有 → 换。

### 8.5 引流三段式（避开导流违禁）

**旧写法（违禁）：** 「加微信 / 私信我 / 我发你资料 / 领课 / 链接放主页」

**新写法（合规三段式）：**

```
[替观众说心里话] + [系列悬念埋钩] + [具体选项评论 + 软性关注]
```

**参考模板：**
> 我知道你现在心里肯定在想——听着挺爽，我该怎么上手？别急。这套 skill 我已经用了小半年，一点点打磨出来的。搭建思路、31 套风格的效果对比、还有一些踩过的坑，我陆陆续续都会拆开讲。评论区告诉我，你平时卡在剪视频的哪一步最崩溃——是对字幕对到眼花，还是选不出风格，还是配画面没灵感？你说得越具体，我下一条就越对你的胃口。想追这个系列的朋友，顺手关注一下，别错过就好。

**拆解：**
1. **替观众说心里话**「听着挺爽，我该怎么上手？」— 精准命名
2. **系列悬念埋钩**「搭建思路 / 效果对比 / 踩过的坑，陆陆续续拆开讲」— 勾住看后续
3. **具体选项评论**「对字幕、选风格、配画面」— 给具体选项比开放问题回复率高
4. **软性关注**「顺手关注一下，别错过就好」— 弱语气，不硬推

**禁用引流词：**
- ❌ 加微信 / 私信 / 加我
- ❌ 链接 / 领取 / 资料我整理好了（后半句"资料"也不要提，容易被判引流）
- ❌ 主页 / 简介 / 置顶（引导站外交易的常见前缀）
- ✅ 评论区告诉我 / 顺手关注一下 / 先收藏

### 8.6 精简稿手法（详细稿 → 精简稿）

**规则：详细稿是画面规划稿，精简稿是录音稿。** 精简时按以下手法压缩，不是逐句删。

| 手法 | 详细稿写法 | 精简稿写法 |
|------|-----------|-----------|
| **删修饰前缀** | 用法真的简单到有点离谱 | 用法就三样东西 |
| **删重复画面描写** | 文字跳出来、颜色铺开、画面一帧一帧长出来，你什么都不用管 | AI 直接在后台把字变成带动画的画面 |
| **合并同意群** | 场景07「适合什么人」+ 场景08「三类人」 | 合成一句「三类人特别适合：…」 |
| **保留数字锚点** | 3 套布局、31 套视觉风格 | 3 套布局、31 套视觉风格 ✅ 数字必须留 |
| **保留具体动作** | 天没亮开机，天黑了还盯着轨道发呆，脖子僵得转不动 | 天没亮开机，天黑了还盯着轨道 ✅ 核心画面留 |
| **删情绪展开** | 一年下来，别人还在起步，你已经跑出去很远了 | ✂️ 删掉（结论前一句已给完） |
| **保留悬念钩** | 我该怎么上手？别急。这套 skill 我已经用了小半年… | ✂️ 精简稿只保留互动指令，悬念钩留给详细稿 |

**核心原则：**
- 数字/核心画面/关键对比 → 保留
- 铺垫词/情绪展开/嵌套修饰 → 删
- 场景切换句和金句可以合并
- 字数控制在详细稿的 40%-50%

### 8.7 稿件生成的两遍法流程（v2）

**每次生成 text-motion-skill 稿件，走这个流程，不要跳步：**

```
Step 1  确认类型 → 类型 A / B（用附页快速通道判断）
Step 2  生成 3 秒爆款开头（按 8.1 三合一公式）
Step 3  写详细稿正文（覆盖四层价值框架）
Step 4  过 8.2 术语扫描（用户听不懂就换）
Step 5  过 8.4 画面感清单（形容词换场景）
Step 6  加 8.5 引流段（三段式合规）
Step 7  过 8.3 隐形极限词扫描
Step 8  加场景标注 → 生成 script-detailed.md
Step 9  用 8.6 精简手法压缩 → 生成 script-compressed.md
Step 10 双稿字数比：精简稿 ≈ 详细稿 40%-50%
```

**输出目录规范：**

```
projects/YYYYMMDD-<slug>/
  ├── script-detailed.md    # 详细稿（分镜规划，含场景标注）
  ├── script-compressed.md  # 精简稿（录音用，无场景标注）
  ├── audio.mp3             # 录制后放这里
  ├── audio.srt             # 转录后放这里
  └── scene-timing.json     # 从 SRT 自动生成，锚点匹配详细稿场景
```

**同时可选发布通用版到 `自媒体文章/` 目录**（用作公众号/短视频通稿）。
