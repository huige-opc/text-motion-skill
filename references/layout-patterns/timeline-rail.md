# timeline-rail 时间轴导轨布局

用于多个字幕 beat、流程、步骤、先后顺序。比普通卡片网格更适合填满横向画面，并能降低"PPT 卡片堆叠感"。

## 结构

- 横向或纵向导轨贯穿画面 55%-72% 宽度。
- 每个 beat 对应一个 node，最多 3-4 个。
- 当前重点 node 可配一个详情面板，其他 node 保持轻量。

## 规则

- node 不要平均堆在左侧；导轨应跨过画面中线。
- 每个 node 文案不超过 18 个中文字符。
- 详情面板最多 2 行正文。
- 装饰线条服务阅读顺序，不做无意义复杂电路图。

## 常见用法

- 讲步骤：输入 → 分析 → 生成 → 复盘。
- 讲时间点：第 3 秒、第 5 秒、第 10 秒发生什么。
- 讲系统流程：用户需求 → agent 执行 → 网页 → MP4 输出。
