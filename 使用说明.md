# 文字动画视频 skill（v3）

## 这个 skill 做什么

**给我一个主题（或 SRT），我给你一个"网页版视频"**：

- **独立网页播放器** — 打开 `index.html` 就能播
- **RAF 咬合齿轮** — 每帧读 `audio.currentTime`，画面跟着口播节奏出场
- **可选导出 MP4** — headless Chrome + CDP + FFmpeg

## 三个入口模式

| 模式 | 适用 | 素材 |
|------|------|------|
| **全屏**（fullscreen） | AI 写稿+配音，无人像 | 不需要 |
| **人像居右**（portrait-right） | 用户自己口播，人像在右 | 需 `portrait.mp4` |
| **人像居中**（portrait-center） | 用户自己口播，人像居中全屏 | 需 `background.mp4` |

技能触发后自动引导选择：布局 → 风格主题 → 内容来源。

## 包内文件

- `SKILL.md`：完整技能协议
- `scripts/`：11 个内置脚本（初始化、TTS、SRT 匹配、MP4 渲染、项目检查）
- `templates/`：3 套布局模板（fullscreen / portrait-right / portrait-center）
- `themes/`：31 个风格主题
- `references/`：场景规范、口播稿规则、审美守则、组件库
- `projects/`：项目文件在这里自动创建

## 项目位置

初始化命令跑完后，项目自动创建在 `text-motion-skill/projects/` 下。命名规则：

```
projects/
├── 20260721/            ← 当天第一个项目
├── 20260721-1/          ← 当天第二个
└── 20260721-2/          ← 当天第三个
```

用户的口播录音、人像视频等素材放到 `{项目根目录}/`。

## 自动化 MP4 依赖

首次使用：

```powershell
cd d:\AI_bian_cheng\trae\skills\text-motion-skill
npm i chrome-remote-interface
```

系统还需要：Chrome 浏览器、FFmpeg（在 PATH 里）、Node.js 22+

渲染：

```powershell
node "{[skillRoot]}\scripts\render-mp4.mjs" "{[projectRoot]}"
```

## 注意事项

- 口播稿不问时长，内容结构自然决定节奏
- 全屏模式 AI 写稿+配音，人像模式用户视频提取音频+SRT
- 时间轴真相来自 SRT 时间戳，禁止估算等距切割
- HTTP 服务器必须支持 Range 请求（禁止 python -m http.server）
- 所有场景写完跑 `node scripts/check-project.mjs {项目根目录}`
